LESSON 01 / 23

Node.js 的角色

JavaScript 脫離瀏覽器後的完整運作原理，以及 AI Agent 是如何調用它來執行你的專案。

1 JavaScript 的兩條世界線

JavaScript 原本是一種「瀏覽器內嵌語言」——它只能在 Chrome、Safari 裡跑。直到 2009 年，Ryan Dahl 把 Google Chrome 的 V8 JavaScript 引擎 抽出來，包了一層 C++ 外殼，讓 JavaScript 能直接在作業系統上執行。這就是 Node.js。

這意味著同一套 JavaScript 語法，現在可以在兩個地方運行：

瀏覽器
// 受限環境
DOM、fetch、localStorage
無法讀檔案系統
無法開啟網路端口
Node.js
// 完整系統權限
readFile、writeFile
開啟 HTTP server
執行 shell 命令
2 AI Agent 調用 Node.js 的完整流程

當你對 Claude Code 說：「幫我建立一個 Next.js 專案，加入登入功能」，它實際上經歷了以下步驟：

$ npx create-next-app@latest my-app
✔ What is your project named? ... my-app
✔ Would you like to use TypeScript? ... No
✔ Would you like to use Tailwind CSS? ... Yes
$ cd my-app && npm run dev
▲ Next.js dev server running on http://localhost:3000
Ready in 1.2s

npx 本質上就是讓 Node.js 下載並執行一個工具。npm run dev 則是啟動 Next.js 的開發伺服器——它背後跑的是一個 Node.js HTTP Server。

3 深入理解
V8 引擎：JavaScript 的發動機

Chrome 瀏覽器內建一個叫做 V8 的 JavaScript 引擎，負責把 JS 程式碼編譯成電腦能執行的機器碼。V8 的速度非常快，因為它使用即時編譯（JIT Compilation）技術——邊解釋邊優化。

Node.js 的做法很聰明：不重新發明輪子，直接借用 V8，再加上檔案系統、網路、進程管理等作業系統 API。這就是為什麼 Node.js 能同時處理大量連線——它不需要為每個連線開一個新執行緒。

Event Loop：單執行緒如何處理所有事情？

Node.js 的核心是 事件循環（Event Loop）。想像一個餐廳服務生：他只有一雙手（單執行緒），但他不需要等每一道菜做完才去下一桌——他把訂單遞給廚房（非同步 I/O），然後立刻去服務下一位客人。

當廚房做好菜（I/O 完成），會通知服務生去送菜。這個「通知」機制就是 Callback / Promise / Async-Await。

$ node -e "console.log('開始'); setTimeout(() => console.log('2秒後'), 2000); console.log('結束');"
開始
結束
2秒後

注意「結束」出現在「2秒後」前面——這就是非同步的魔力。

Port 衝突：EADDRINUSE 是什麼意思？

每台電腦有 65,535 個連接埠（Port）。當 Node.js 啟動伺服器時，它需要「佔用」一個 Port 來對外服務。如果兩個程式都想用 3000，就會噴出：

Error: listen EADDRINUSE: address already in use :::3000

解決方法很簡單：換一個 Port。npm run dev -- -p 3001 就能讓第二個專案用 3001 號房。

✏️ 小測驗

請先回答，再點擊「查看答案」

1. Node.js 的本質是什麼？
  • A. 一種新的程式語言
  • B. 把 V8 引擎帶出瀏覽器的執行環境
  • C. 瀏覽器內建的 JavaScript 引擎
✅ 正確答案：B。Node.js 是 Ryan Dahl 在 2009 年把 Chrome 的 V8 引擎抽出來，讓 JS 能在作業系統上直接執行。
2. 當兩個 Node.js 伺服器都嘗試使用 Port 3000 時，會發生什麼事？
  • A. 第二個伺服器會報 EADDRINUSE 錯誤
  • B. 兩個伺服器會自動平分流量
  • C. Node.js 會自動關閉第一個伺服器
✅ 正確答案：A。Port 就像房間號碼，同一時間只能被一個程序佔用。
3. Event Loop 讓 Node.js 能夠：
  • A. 同時使用多個 CPU 核心
  • B. 單執行緒處理大量非同步 I/O 操作
  • C. 自動修復程式碼中的 Bug
✅ 正確答案：B。Node.js 用單執行緒 + 非同步 I/O 來高效處理大量連線。
