# Neumorphism 設計系統 - 教學 App

## 設計理念

Neumorphism（新擬態設計）是一種現代 UI 設計風格，通過精細的陰影效果（包含外陰影和內陰影）來模擬浮雕或嵌入的立體效果。這種設計風格創造出柔和、溫暖且具觸感的視覺體驗，非常適合教育類應用。

## 色彩系統

### 主要色彩

| 色彩用途 | 色值 | 說明 |
|---------|------|------|
| 背景色 | `RGB(237, 240, 244)` | 柔和的淺灰色，營造舒適閱讀體驗 |
| 深色模式背景 | `RGB(61, 64, 69)` | 深沉的中性色，保護眼睛 |
| 主要文字 | `RGB(26, 26, 46)` | 深色但不純黑，減少對比刺激 |
| 次要文字 | `RGB(115, 119, 128)` | 柔和的灰色，層次分明 |
| 強調色 | `RGB(102, 126, 234)` | 清新的藍紫色，充滿學術氣息 |
| 強調色漸層 | `#667eea → #764ba2` | 漸變效果增加視覺深度 |
| 成功色 | `RGB(5, 150, 105)` | 柔和的綠色，代表完成 |
| 警告色 | `RGB(245, 158, 11)` | 溫暖的橙色，引起注意 |
| 錯誤色 | `RGB(220, 38, 38)` | 鮮明的紅色，標示錯誤 |

### 陰影系統

#### 凸起效果（凸出表面）
```css
box-shadow: 
  8px 8px 16px rgba(0, 0, 0, 0.12),
  -8px -8px 16px rgba(255, 255, 255, 0.75);
```

#### 凹陷效果（嵌入表面）
```css
box-shadow: 
  inset 4px 4px 8px rgba(0, 0, 0, 0.15),
  inset -4px -4px 8px rgba(255, 255, 255, 0.7);
```

#### 按下效果（按壓狀態）
```css
box-shadow: 
  2px 2px 4px rgba(0, 0, 0, 0.15),
  -2px -2px 4px rgba(255, 255, 255, 0.7);
```

## 主要組件

### 1. Neumorphic Card（新擬態卡片）

**用途**: 課程卡片、內容容器

**特徵**:
- 背景色與頁面背景相同
- 柔和的外陰影創造浮起效果
- 圓角 20px
- 內邊距 20px

**使用場景**:
- 課程列表項目
- 資訊展示區塊
- 功能入口

### 2. Neumorphic Button（新擬態按鈕）

**用途**: 主要操作、導航

**變體**:
- **Primary**: 漸層背景，白色文字，帶陰影
- **Secondary**: 凹凸效果，文字+圖標
- **Pill**: 胶囊形狀，漸層填充
- **Circle**: 圓形圖標按鈕

**互動狀態**:
- 默認：凸起效果
- 懸停：陰影加深
- 按下：凹陷效果

### 3. Neumorphic Toggle（新擬態開關）

**用途**: 設置開關、狀態切換

**特徵**:
- 凹槽軌道
- 圓形滑塊帶陰影
- 平滑動畫過渡
- 顏色變化指示狀態

### 4. Neumorphic Input（新擬態輸入框）

**用途**: 文字輸入、搜索

**特徵**:
- 凹陷效果
- 清晰的邊界
- 聚焦時強調色邊框
- 足夠的內邊距確保可讀性

### 5. Neumorphic Progress（新擬態進度條）

**用途**: 學習進度、載入狀態

**變體**:
- **圓形進度**: 帶百分比數字
- **線性進度**: 漸層填充軌道

### 6. Neumorphic Tab Bar（新擬態標籤欄）

**用途**: 主要導航

**特徵**:
- 凹槽背景
- 活動項目標記
- 平滑過渡動畫
- 底部固定定位

## 動畫與過渡

### 時機

| 動畫類型 | 持續時間 | 曲線 |
|---------|---------|------|
| 微交互 | 150-200ms | Ease-out |
| 狀態切換 | 200-300ms | Spring |
| 頁面轉換 | 300-400ms | Ease-in-out |

### 常見動畫

1. **按鈕按下**: 縮放 0.95 → 1.0，陰影變化
2. **卡片浮現**: 從下方滑入 + 淡入
3. **開關切換**: 滑塊移動 + 顏色漸變
4. **頁面切換**: 橫向滑動 + 淡入淡出

## Dark Mode 支援

### 色彩映射

| 元素 | Light Mode | Dark Mode |
|------|-----------|-----------|
| 背景 | `#EDEFF4` | `#3D4045` |
| 凸起陰影（暗） | `rgba(0,0,0,0.12)` | `rgba(0,0,0,0.4)` |
| 凸起陰影（亮） | `rgba(255,255,255,0.75)` | `rgba(255,255,255,0.05)` |
| 文字 | `#1A1A2E` | `#FFFFFF` |
| 次要文字 | `#737780` | `#B2B4B8` |

### 注意事項

- Dark Mode 下陰影對比度更高
- 確保文字對比度符合 WCAG 標準
- 測試兩種模式下的可讀性

## 設計原則

### 必須遵守 ✓

1. **一致的陰影方向**: 所有元素使用相同的陰影光源方向
2. **相同的背景色**: 組件背景必須與頁面背景一致
3. **足夠的對比度**: 文字與背景對比度 ≥ 4.5:1
4. **合理的間距**: 遵循 8pt 網格系統
5. **平滑的動畫**: 所有狀態切換都要有過渡動畫

### 避免避免 ✗

1. **不要混用風格**: 保持 Neumorphism 風格一致性
2. **避免純黑色**: 使用深灰色代替
3. **避免過重的陰影**: 陰影應該柔和自然
4. **不要忽略觸控目標**: 最小觸控區域 44×44pt
5. **避免複雜的背景**: 保持單色背景

## 響應式設計

### 斷點

| 設備類型 | 寬度 | 間距調整 |
|---------|------|---------|
| 手機（小） | < 375px | 減少 20% |
| 手機（標準） | 375-414px | 標準 |
| 平板 | > 768px | 增加 20% |

### 自適應策略

1. **卡片布局**: 手機單欄，平板雙欄
2. **字型大小**: 使用相對單位，支持動態類型
3. **間距**: 隨螢幕尺寸按比例調整
4. **圖標大小**: 保持可識別的最小尺寸

## 无障碍設計

### 對比度要求

- 普通文字: ≥ 4.5:1
- 大文字: ≥ 3:1
- 圖標和界面組件: ≥ 3:1

### 輔助功能

1. **鍵盤導航**: 所有交互元素可鍵盤訪問
2. **屏幕閱讀器**: 提供有意義的標籤
3. **動態類型**: 支持系統字型大小調整
4. **減少動態效果**: 尊重用戶的減少動態設置
