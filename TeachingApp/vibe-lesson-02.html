LESSON 02 / 23

Localhost 與 Port（連接埠）

電腦公寓理論，解決 EADDRINUSE 打架衝突，理解為什麼你的 Vibe 專案需要「自己的房間」。

1 什麼是 Localhost？

Localhost 其實就是你的電腦自己——它的 IP 地址是 127.0.0.1。這不是一個神秘的網路位址，而是作業系統保留給「自我連線」的專屬門牌。

當你對 AI Agent 說「幫我啟動開發伺服器」，它跑出來的網址通常是：

▲ Next.js dev server running on http://localhost:3000
Ready in 1.2s

localhost = 127.0.0.1 = 你自己。:3000 = Port 3000，也就是這棟公寓裡的第 3000 號房間。

⚡ 為什麼多個任務會打架？

每台電腦有 65,535 個連接埠（Port 0~65535）。每個 Port 就像一個獨立的房間，同一時間只能住一個程序。

當你在兩個不同的 Vibe 專案裡都執行 npm run dev，它們都想擠進 3000 號房，於是第二個專案就噴出：

Error: listen EADDRINUSE: address already in use :::3000
// EADDRINUSE = Address Already In Use

這就是 Port 衝突。AI Agent 在幫你同時開多個專案時，很容易遇到這個問題。

3 深入理解
如何正確地「換房」？

不同框架有不同的指定 Port 方式：

# Next.js — 使用 --port 參數
$ npx next dev --port 3001
# Vite — 使用 --port 參數
$ npm run dev -- --port 3002
# Express — 設定 PORT 環境變數
$ PORT=3003 node server.js

記住：AI Agent 幫你開專案時，你可以直接告訴它「用 port 3008」，避免衝突。

如何查看哪些 Port 正在被佔用？

在 macOS / Linux 上，可以用 lsof 或 netstat 查看：

# 查看誰佔用了 port 3000
$ lsof -i :3000
# 查看所有監聽中的 Port
$ lsof -iTCP -sTCP:LISTEN

如果看到不需要的程序，可以用 kill <PID> 關閉它。

為什麼 localhost 不能用其他電腦存取？

localhost 永遠只連到自己。這是作業系統的網路規則，跟 Port 無關。

如果你想讓同一個 Wi-Fi 下的其他裝置也能存取你的開發伺服器，需要使用你電腦的 區域 IP（例如 192.168.1.100:3000）。

# 查看自己的區域 IP
$ ipconfig | findstr IPv4
IPv4 Address. . . . . . . . . . . : 192.168.1.100

1. EADDRINUSE 錯誤代表什麼意思？
  • A. 網路斷線
  • B. Port 已被其他程式佔用
  • C. Node.js 版本太舊
✅ 正確答案：B。EADDRINUSE = Address Already In Use，代表你要使用的連接埠已經被另一個程式佔用了。
2. localhost 的 IP 地址是什麼？
  • A. 127.0.0.1
  • B. 192.168.0.1
  • C. 0.0.0.0
✅ 正確答案：A。127.0.0.1 是作業系統保留給自我連線的專屬 IP。
3. 一台電腦總共有多少個連接埠？
  • A. 1,000 個
  • B. 10,000 個
  • C. 65,535 個
✅ 正確答案：C。Port 編號範圍是 0~65535，共 65,536 個位置。
