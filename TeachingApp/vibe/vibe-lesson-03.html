Phase 1: The Core

課程第 3 課：認識 HTTP 請求與回應

理解 AI Agent 如何透過網路協議與你的 Node.js 伺服器對話

🌐 HTTP 是什麼？Vibe Coding 的溝通語言

在 Vibe Coding 中，Node.js 伺服器就像是一個服務生，而 HTTP（HyperText Transfer Protocol）則是客人（瀏覽器或 AI Agent）與服務生之間的點餐單。AI Agent 不會直接「寫入」你的記憶體，而是發送一個 HTTP 請求（例如 GET 或 POST），伺服器處理後回傳一個 HTTP 回應（包含狀態碼和資料）。這是前端與後端互動的基礎。

🔬 深入解析：AI Agent 的調用流程

當你在 Vibe Coding 環境中指令 AI '顯示使用者列表' 時，AI Agent 內部會模擬一個 fetch() 或 axios.get() 呼叫，指向你的 localhost:3000/users。它期待收到一個 200 OK 狀態碼以及 JSON 格式的數據。如果伺服器沒啟動、路徑錯誤，或回傳了 HTML 而非 JSON，Vibe 就會破裂。

// AI Agent 模擬的請求 GET http://localhost:3000/api/status HTTP/1.1 Host: localhost:3000 // 正確的伺服器回應 (200 OK) HTTP/1.1 200 OK Content-Type: application/json {"status": "healthy", "version": "1.0"}
🛠️ Vibe 破裂診斷：常見錯誤碼解讀

當 Vibe 破裂，首先檢查 AI Agent 報錯的 HTTP 狀態碼：

• 404 Not Found：路徑寫錯了，或 Node.js 沒有設定對應的 app.get()。
• 500 Internal Server Error：伺服器程式碼崩潰，請檢查 Node.js 終端的堆疊追蹤（Stack Trace）。
• CORS Error：跨域問題，通常發生在前端與後端端口不一致時，需安裝 cors 套件。

✏️ 小測驗

請先回答，再點擊「查看答案」

1. 如果 AI Agent 發送請求後收到 404 錯誤，最可能的原因是什麼？
  • A. 伺服器已經關機
  • B. 請求的路徑在伺服器上不存在或未定義
  • C. 資料庫連線失敗
✅ B. 404 代表找不到資源，通常意味著路由（Route）未正確配置。
🚀 今日任務

請依序完成以下練習

  1. 使用 Node.js 內建的 http 模組或 Express，建立一個簡單的 GET 路由 /hello，回傳 JSON {"message": "Hello Vibe"}。
  2. 模擬 AI Agent 使用 curl 或 Postman 發送請求，確認收到 200 狀態碼與正確 JSON，若收到 404 請檢查路由設置。
