Phase 1: The Core

第 4 課：理解 HTTP 狀態碼與錯誤處理

從 200 OK 到 404 Not Found，學會聽懂伺服器的語言

📡 為什麼需要狀態碼？

在 Vibe Coding 中，當你的 AI Agent 發送一個 HTTP 請求後，伺服器不會只回傳「成功」或「失敗」，而是會給出一個數字代號，稱為 HTTP 狀態碼。這就像交通號誌：2xx 代表順利通行（成功），4xx 代表客戶端犯錯（例如找不到檔案），5xx 代表伺服器內部崩潰。AI Agent 依賴這些代碼來決定下一步該繼續執行還是回報錯誤。

🔬 深入解析：常見狀態碼與 Vibe 破裂診斷

當 Vibe 破裂時，90% 的情況是因為狀態碼不符合預期。AI Agent 通常會檢查回應中的狀態碼。如果它期待 200 OK 卻收到 404 Not Found，Agent 會知道路徑寫錯了；若收到 500 Internal Server Error，則表示後端程式碼有 Bug。診斷關鍵：不要只看文字訊息，先看狀態碼分類，能快速縮小排查範圍。

GET /api/users/999 HTTP/1.1 404 Not Found Content-Type: application/json { "error": "User not found" }
✏️ 小測驗

請先回答，再點擊「查看答案」

1. 如果你的 AI Agent 試圖存取一個不存在的 API 端點，伺服器回傳的狀態碼最可能是多少？
  • A. 200 OK
  • B. 404 Not Found
  • C. 500 Internal Server Error
✅ 404 Not Found 是標準的客戶端錯誤代碼，表示伺服器找不到請求的資源。
🚀 今日任務

請依序完成以下練習

  1. 使用 Postman 或瀏覽器開發者工具，故意輸入一個錯誤的 URL，觀察並記錄回傳的狀態碼。
  2. 修改你的 Node.js 程式碼，讓它在發生錯誤時明確回傳 404 狀態碼，而非預設的 200。
