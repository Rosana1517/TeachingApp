Node.js 的角色

JavaScript 脫離瀏覽器後的完整運作原理，以及 AI Agent 是如何調用它來執行你的專案。

1 JavaScript 的兩條世界線

JavaScript 原本是一種「瀏覽器內嵌語言」——它只能在 Chrome、Safari 裡跑。直到 2009 年，Ryan Dahl 把 Google Chrome 的 V8 JavaScript 引擎 抽出來，包了一層 C++ 外殼，讓 JavaScript 能直接在作業系統上執行。這就是 Node.js。

瀏覽器端

Chrome / Safari
操作 DOM、CSS、事件
受限於安全沙箱

伺服器端

Node.js + V8 引擎
讀寫檔案、連資料庫
擁有系統級權限

2 AI Agent 調用 Node.js 的完整流程

當你對 Claude Code 說：「幫我建立一個 Next.js 專案，加入登入功能」，它實際上經歷了以下步驟：

Step 1 — 建立專案結構：執行 npx create-next-app@latest my-project

Step 2 — 安裝依賴：執行 npm install better-auth，將套件寫入 package.json 並下載到 node_modules

Step 3 — 寫入代碼：直接編輯你的 .ts/.tsx 檔案

Step 4 — 啟動開發伺服器：執行 npm run dev，Node.js 啟動 Next.js 伺服器，監聽 Port 3000

Step 5 — 讀取錯誤：如果報錯，AI 會分析終端機輸出的 Stack Trace，然後修正代碼

🔬 深入：V8 引擎到底是怎麼工作的？

V8 是 Google 用 C++ 寫的開源 JavaScript 引擎，也是 Chrome 瀏覽器的核心。它的任務是把人類寫的 JavaScript 原始碼轉換成 CPU 能執行的機器碼。

Parser（解析器） → 拆解成抽象語法樹（AST）
Ignition（解釋器） → 逐行翻譯成位元組碼（Bytecode）
TurboFan（編譯器） → 偵測 Hot Path，編譯成高度優化的機器碼
GC（垃圾回收器） → 自動清理不再使用的記憶體

這代表 Node.js 的效能之所以快，是因為 V8 不只是「解釋」程式碼，還會在執行過程中動態優化。

🧠 深入：Event Loop —— Node.js 的單執行緒秘密

Node.js 是單執行緒的。當它讀取一個檔案時，不會「等」檔案讀完，而是把讀取任務丟給作業系統，自己回頭處理其他請求。檔案讀完後，結果放進 Callback Queue，Event Loop 會在 Call Stack 空閒時取出執行。

Call Stack → Web APIs / OS APIs → Callback Queue → Call Stack 有空時取出執行

這就是為什麼 Node.js 適合 I/O 密集型應用（API、資料庫查詢），但不適合大量計算。

📦 深入：CommonJS vs ESM —— Node.js 的模組系統之爭
CommonJS
const fs = require('fs');
module.exports = myFunc;
ESM
import fs from 'fs';
export default myFunc;

如果你看到 require() 和 import 混用導致報錯，這就是模組系統的衝突。AI 通常會根據專案配置自動選擇正確的語法。

⚙️ 深入：PATH 環境變數與 .env 安全規範

當你在終端機輸入 node，作業系統會去一串叫做 PATH 的路徑清單裡搜尋可執行檔。如果出現 node: command not found，代表 Node.js 的路徑沒有被正確設定。

.env 安全規範：API Key 不能寫死在代碼裡推上 GitHub。

❌ 錯誤
const key = "sk_live_xxx";
✅ 正確
STRIPE_KEY=sk_live_xxx
process.env.STRIPE_KEY

✏️ 小測驗

1. Node.js 的本質是什麼？
  • A. 一種新的程式語言
  • B. 把 V8 引擎帶出瀏覽器的執行環境
  • C. 瀏覽器內建的 JavaScript 引擎
✅ 正確答案：B。Node.js 是 Ryan Dahl 在 2009 年把 Chrome 的 V8 引擎抽出來，讓 JS 能在作業系統上直接執行。
2. 當你對 AI 說「幫我安裝套件」，它底層執行的是哪個命令？
  • A. pip install
  • B. cargo add
  • C. npm install
✅ 正確答案：C。NPM 隨 Node.js 一起安裝，是 AI 用來管理第三方套件的管家。
3. 為什麼 API Key 不能寫在代碼裡？
  • A. 會被推上 GitHub 洩漏給所有人
  • B. 會影響程式執行速度
  • C. Node.js 無法讀取字串變數
✅ 正確答案：A。敏感資訊應放在 .env 中，並透過 .gitignore 排除在版本控制之外。

🚀 今日任務：檢查你的 Node.js 環境

  1. 打開終端機，依序執行 node -v、npm -v、npx -v
  2. 確認三個命令都有輸出版本號
  3. 進階挑戰：執行 node -e "console.log('Hello from Node.js'); setTimeout(() => console.log('2秒後'), 2000);"，體驗 Event Loop
