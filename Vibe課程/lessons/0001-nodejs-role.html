Node.js 的角色

JavaScript 為什麼能離開瀏覽器，在你的電腦上直接執行？

JavaScript 原本只能待在瀏覽器裡

JavaScript 誕生於 1995 年，設計初衷只是讓網頁能互動——按鈕點擊、表單驗證。它一直被關在 Chrome、Safari 這些瀏覽器的「安全沙箱」裡，不能讀取你的硬碟、不能連資料庫、不能架設伺服器。

直到 2009 年，Ryan Dahl 做了一件事：他把 Google Chrome 裡的 V8 JavaScript 引擎 抽出來，包上一層 C++ 外殼，讓 JavaScript 能直接在作業系統上執行。這就是 Node.js。

同一個語言，兩個世界

瀏覽器端

Chrome / Safari
操作 DOM、CSS、事件
受限於安全沙箱

伺服器端

Node.js + V8 引擎
讀寫檔案、連資料庫
擁有系統級權限

AI Agent 怎麼用到 Node.js？

當你對 Claude Code 說「幫我安裝一個套件」時，它底層做的事情只有一個：npm install。

$ npm install date-fns
added 1 package in 2.3s

NPM（Node Package Manager） 隨 Node.js 一起安裝。它是全球最大的開源套件倉庫，裡面有超過 200 萬個套件。沒有 Node.js，AI 就沒有「手」來幫你下載和管理這些工具。

🔬 深入：V8 引擎如何把程式碼變成機器能懂的指令？

V8 是 Google 用 C++ 寫的開源 JavaScript 引擎。它的工作是把人類寫的 JavaScript 原始碼轉換成 CPU 能執行的機器碼。

原始碼 → Parser（拆解成 AST）→ Ignition（解釋成 Bytecode）→ TurboFan（優化為機器碼）→ GC（清理記憶體）

這代表 Node.js 的效能之所以快，是因為 V8 不只是「解釋」程式碼，還會在執行過程中動態優化被頻繁呼叫的區域。

🧠 深入：Event Loop —— Node.js 為什麼是「單執行緒」卻不卡住？

Node.js 是單執行緒的。當它讀取一個檔案時，不會傻等檔案讀完，而是把任務丟給作業系統，自己回頭處理其他請求。檔案讀完後，結果放進 Callback Queue，Event Loop 會在 Call Stack 空閒時取出執行。

Call Stack → OS API → Callback Queue → 有空時取出執行

這就是為什麼 Node.js 適合 API 服務和資料庫查詢，但不適合大量數學運算。

✏️ 小測驗

Node.js 的本質是什麼？
  • 一種全新的程式語言
  • 把 V8 引擎帶出瀏覽器的執行環境
  • 瀏覽器內建的 JavaScript 引擎
正確答案：把 V8 引擎帶出瀏覽器的執行環境。Ryan Dahl 在 2009 年把 Chrome 的 V8 引擎抽出來，讓 JS 能在作業系統上直接執行。
AI 幫你安裝套件時，底層執行的是哪個命令？
  • pip install
  • npm install
  • cargo add
正確答案：npm install。NPM 隨 Node.js 一起安裝，是 AI 用來管理第三方套件的管家。

🚀 今日任務

  1. 打開終端機，執行 node -v，確認有輸出版本號
  2. 執行 npm -v，確認 NPM 也隨 Node.js 一起安裝了
  3. 進階挑戰：執行 node -e "console.log('Hello from Node.js'); setTimeout(() => console.log('2秒後'), 2000);"，體驗 Event Loop 的非同步行為
